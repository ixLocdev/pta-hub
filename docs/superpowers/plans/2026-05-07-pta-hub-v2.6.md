# PTA Knowledge Hub v2.6 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship six new features across two releases — reading time + TOC, QR codes, stale-content reminders, public preview links (v2.6.0); suggest-a-topic form, Wizard related-entries panel (v2.6.1).

**Architecture:** One new class file per feature under `pta-knowledge-hub/includes/`, registered from the plugin bootstrap. Two cross-cutting changes (a `ptk_hub_url()` helper and a public `PTK_Multisite::is_syncing()` accessor) ship first as shared scaffolding. Each release ends with a version bump + changelog.

**Tech Stack:** PHP 7.4+, WordPress 6.0+, no build pipeline, no Composer, no automated test suite. Verification = `php -l` + manual smoke test on the live multisite.

**Spec:** [docs/superpowers/specs/2026-05-07-pta-hub-v2.6-design.md](../specs/2026-05-07-pta-hub-v2.6-design.md)

**Note on testing:** Per user instruction, this plan omits automated tests because the codebase has none. Each task ends with a `php -l` lint check and a documented smoke test the user runs on the live site.

  - `auto_stamp( $post_id, $post )`: bail if `wp_is_post_revision()`, `DOING_AUTOSAVE`, `$post->post_status !== 'publish'`, OR `class_exists('PTK_Multisite') && PTK_Multisite::is_syncing()`. Otherwise `update_post_meta( $post_id, 'ptk_last_reviewed', current_time( 'Y-m-d' ) )`.

  - `handle_mark_reviewed()`: capability `edit_post`; nonce; update meta; `wp_safe_redirect( wp_get_referer() )`.
  - `render_dashboard_widget()`: cap at 20 results, oldest first. Query shape (note the `NOT EXISTS` clause — that's the WP-correct way to find posts where meta is missing entirely):
    ```php
    $threshold_date = date( 'Y-m-d', strtotime( '-' . self::get_threshold() . ' months' ) );
    $args = [
        'post_type'      => 'pta_knowledge',
        'post_status'    => 'publish',
        'posts_per_page' => 20,
        'orderby'        => 'meta_value',
        'meta_key'       => 'ptk_last_reviewed',
        'order'          => 'ASC',
        'meta_query'     => [
            'relation' => 'OR',
            [
                'key'     => 'ptk_last_reviewed',
                'value'   => $threshold_date,
                'compare' => '<',
                'type'    => 'DATE',
            ],
            [
                'key'     => 'ptk_last_reviewed',
                'compare' => 'NOT EXISTS',
            ],
        ],
        'date_query' => [
            // Only flag missing-meta posts if they're older than threshold themselves.
            // (This filter applies to ALL results; for entries that DO have meta, this is harmless
            // since their post_date is older than their meta date which is older than threshold.)
            'before' => $threshold_date,
            'inclusive' => true,
        ],
    ];
    ```
    Render a table with columns Title / Last Reviewed / Actions ("Mark Reviewed" button + "Edit" link). If list empty, show "All entries are up to date."
  - Inline CSS for `.ptk-review-dot` (8px circle, green/amber/red).
